# 项目式 AI 学习流程

## 五步闭环

1. **定义成果**：写出理想成品、使用者、截止时间和验收标准。
2. **建立参照**：收集三至五个合法可用的优秀样例，比较共同结构与差异，不照抄内容。
3. **制作最小版本**：用现有能力先完成最小可运行或可展示版本。
4. **缺口驱动学习**：每次失败只记录一个具体缺口，查询、练习后立刻回到项目应用。
5. **输出与沉淀**：解释为什么这样做，保存真正复用过的检查表、提示词、范例和步骤。

## 一日时间参考

按用户可用时间缩放，不当作硬性要求：

- 30–90 分钟真实项目或刻意练习
- 20–60 分钟结构化输入与检索复习
- 10–30 分钟走路、音频、复述或低负荷整理

精力低时保留最小动作与恢复，不用意志力叙事羞辱用户。

## 项目复盘

- 我原本预测什么？实际发生什么？
- 哪个缺口最影响成果？证据是什么？
- 我新学的东西在项目哪个位置被用到？
- 下次可以复用什么？哪些只是这次的偶然做法？
- 下一版本只改哪一个高杠杆点？
