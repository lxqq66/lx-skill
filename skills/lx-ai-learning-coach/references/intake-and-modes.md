# 学习需求与模式选择

## 三个高价值问题

优先从以下问题选最多三个，不要全部询问：

- 你想用它完成什么？什么表现能说明你真的学会了？
- 你已经做过或学过什么？能给我看一次尝试吗？
- 截止时间和每次可投入时间是多少？
- 这是自由学习、工作任务，还是会被评分且限制外援的作业或考试？
- 你现在更需要我提问引导、快速解释、出题练习，还是陪你做一个项目？
- 目前更像“不知道从哪开始”“理解不了”“会但做不出”“容易忘”还是“缺少坚持”？

## 模式选择表

| 用户状态 | 首选模式 | 第一动作 |
| --- | --- | --- |
| 只说想学某主题 | 引导探索 | 定义真实结果与微型诊断 |
| 完全没有前置概念 | 最小讲解 | 解释一个核心模型后立即预测 |
| 看懂但做不出 | 练一练 | 一道低门槛题定位断点 |
| 会做但说不清 | 当小老师 | 两分钟无资料复述 |
| 工作中要产出 | 项目学习 | 明确成品、标准和最小版本 |
| 临近考试 | 冲刺复习 | 检索测试和错因排序 |
| 焦虑、怕错、无法开始 | 降低启动门槛 | 两分钟暖身或二选一 |

## 学习契约示例

> 你想在两周后独立做出一张数据看板；学会的证据是不用教程完成导入、清洗和三个图表。我们先用项目模式，今天用十分钟做出最小版本并找出第一个知识缺口。

每完成一轮可以重新协商目标和模式，不把初始计划当成不可改变的合同。
