# 教育诊断输出模板

## 1. 本次要解决的问题

- 学生大致年龄或年级：

- 三十天目标：

## 2. 信息质量

### 已知事实

- 只列用户提供的可观察行为、时间、场景和结果。

### 关键未知

- 列出会显著改变判断的二至三个缺口。

### 孩子视角

- 直接表达：
- 若缺失：明确写“目前只有成人视角，以下判断为初步假设”。

## 3. 核心诊断

### 核心矛盾

用一句话说明最值得先改变的系统条件，不使用人格标签。

### 证据链

| 可观察事实 | 可能机制 | 置信度 | 替代解释/待核实 |
| --- | --- | --- | --- |
|  |  | 高/中/低 |  |

### 次要因素

- 最多列二至三个，不把所有维度都说成问题。

## 4. 认知转折

使用以下句式：

> 你现在看到的是___；目前证据更支持___。这不等于___，而意味着我们可以先改变___，用___来验证。

## 5. 需要保护的东西

- 学生的优势、兴趣、关系资源或曾经成功的条件。
- 明确哪些东西不应为了短期结果被牺牲，例如睡眠、安全感或亲子关系。

- 学生行动：
- 家长行动：
- 教师行动（如适用）：
- 开始时间与场景：
- 记录指标：
- 停止或调整条件：

## 7. 复盘

第七天回答：

1. 哪个变化最明显？
2. 哪个判断被证实，哪个被推翻？
3. 冲突、催促或启动时间是否变化？
4. 学生自己的感受是什么？
5. 下一轮继续、调整还是停止什么？

## 8. 升级条件

- 何时需要补充学校信息。
- 何时需要转用 `lx-parent-learning-environment`。
- 何时需要专业心理、医疗、社会保护或紧急支持。
